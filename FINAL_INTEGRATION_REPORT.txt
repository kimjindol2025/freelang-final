3.2 compiler.fl (신규, 471줄)
────────────────────────────────────────────────────────────────────────────────

목적: 전체 컴파일 파이프라인 통합

주요 구조 (3개):
  - CompilerConfig: 컴파일 옵션 (emit_ir, emit_asm, optimize_level, debug)
  - CompileResult: 결과 구조 (success, output, ir_code, errors, time)
  - CompilerState: 파이프라인 상태 관리

공개 API (2개):
  - compile(source_code) - 기본 설정으로 컴파일
  - compile_with_config(source_code, config) - 커스텀 설정

파이프라인 (6단계):
  Step 1: Lexing (JavaScript)
  Step 2: Parsing (JavaScript)
  Step 3: Semantic Analysis (FreeLang)
  Step 4: IR Generation (FreeLang)
  Step 5: Instruction Selection (FreeLang)
  Step 6: Assembly Output (FreeLang)

헬퍼 함수 (7개):
  - parse_source() - 파서 인터페이스
  - analyze_program() - 의미분석 수행
  - generate_ir_from_ast() - IR 생성
  - ir_to_string() - IR 출력
  - contains() - 부분 문자열 검색
  - os_time() - 타이밍 측정

테스트: 3개 모두 ✅ 통과
  - Test 1: let x = 5;
  - Test 2: fn add(...)
  - Test 3: if statement

특징:
  - 에러 처리 포함
  - 성능 측정 (ms)
  - 설정 가능한 최적화
  - 상세한 진단 메시지

5. 통합 아키텍처
================================================================================

파이프라인 흐름:

Source Code (문자열)
    ↓
Step 1-2: Lexing & Parsing (JavaScript)
    ↓ JavaScript AST
[parse_source() - compiler.fl]
    ↓ FreeLang AST (ast.fl)
[analyze_program() - semantic-analyzer.fl]
    ↓ Verified AST + Errors
[generate_ir_from_ast() - ir-generator.fl]
    ↓ IR Instructions (ir-types.fl)
[select_all_instructions() - x86-64-isel.fl]
    ↓ x86-64 Instructions
[generate_asm() - x86-64-isel.fl]
    ↓ Assembly Source Code (문자열)
CompileResult (success, output, messages)

모듈 의존성:

compiler.fl (최상위)
  ├─ ast.fl
  ├─ semantic-analyzer.fl
  │  ├─ symbol-table.fl
  │  └─ type-system.fl
  ├─ ir-generator.fl
  │  └─ ir-types.fl
  └─ x86-64-isel.fl
