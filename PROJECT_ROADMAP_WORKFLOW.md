## 🤖 에이전트 팀 (4명)

### Agent 1: 컴파일러 엔지니어 (Compiler)

**담당**: Phase 1 + Phase 2 코드생성 기초

| 항목 | 내용 |
|------|------|
| **주요 작업** | 의미분석, IR생성, LLVM통합 |
| **구현 파일** | semantic-analyzer.fl, ir-generator.fl |
| **예상 LOC** | 3000-4000줄 |
| **일정** | 12주 |
| **의존성** | parser.fl (완료) |
| **산출물** | FreeLang → LLVM IR 변환기 |

**체크리스트**:
- [ ] 의미분석기 (변수 타입, 스코프)
- [ ] IR 중간코드 생성
- [ ] LLVM C API 바인딩
- [ ] 단위 테스트 (50+)
- [ ] 성능 벤치마크

### Agent 2: 코드생성 엔지니어 (CodeGen)

**담당**: Phase 2 + Phase 3 기계어 생성

| 항목 | 내용 |
|------|------|
| **주요 작업** | x86-64 생성, 링커 기초 |

**체크리스트**:

- [ ] 레지스터 할당 알고리즘

- [ ] 단순 링커 (심볼 해석)
- [ ] 실행파일 생성 (ELF)

### Agent 3: 런타임 엔지니어 (Runtime)

| 항목 | 내용 |
|------|------|
| **주요 작업** | 메모리 관리, 런타임 시스템 |

| **산출물** | 독립 메모리 관리자 |

**체크리스트**:
- [ ] Malloc/Free 구현
- [ ] Mark-and-Sweep GC
- [ ] 스택 관리
- [ ] 함수 호출 규약
- [ ] 예외 처리 시스템

---

### Agent 4: 최적화 엔지니어 (Optimizer)

**담당**: Phase 2 중반 + 연속적 최적화

| 항목 | 내용 |
|------|------|
| **주요 작업** | 컴파일러 최적화, 성능개선 |

| **예상 LOC** | 1500-2500줄 |

| **의존성** | IR 생성 |

**체크리스트**:

- [ ] 데드 코드 제거 (DCE)
- [ ] 인라이닝 (Inlining)
- [ ] 루프 최적화
- [ ] 성능 비교 (Rust vs FreeLang)

## 📅 Phase별 세부 계획

### Phase 1: 컴파일러 기초 (Weeks 1-12)

#### Week 1-2: 설계 및 의존성 분석

**목표**: 아키텍처 설계 완료, 테스트 프레임워크 구축

**담당**: 팀장(Claude) + Compiler Agent

**작업**:
```
❌ → ✅ 체크리스트:
- [ ] 전체 아키텍처 설계 (AST → IR)
- [ ] LLVM 바인딩 분석
- [ ] 의존성 맵핑
- [ ] 테스트 케이스 200개 작성
- [ ] FreeLang 테스트 프레임워크 구축
```

**산출물**:
- architecture-design.md (20줄)
- compiler-plan.fl (150줄)
- test-framework.fl (200줄)

**검증**: 팀장 기술리뷰

---

#### Week 3-8: 의미분석기 + IR 생성

**목표**: FreeLang 코드 → LLVM IR 변환 완성

**담당**: Compiler Agent + Runtime Agent (병렬)

**Compiler 작업**:
```
Step 1: 의미분석기 (semantic-analyzer.fl)
  ├── 변수 선언 추적
  ├── 함수 시그니처 검증
  ├── 타입 불일치 감지
  └── 스코프 관리

Step 2: IR 생성 (ir-generator.fl)
  ├── 3-주소 코드
  ├── 기본 블록 생성
  ├── 제어 흐름 그래프
  └── LLVM IR 변환
```

**Runtime 병렬 작업**:
```
메모리 모델 설계
  ├── 힙 레이아웃
  ├── 스택 프레임
  ├── GC 마킹 스킴
  └── 런타임 지원
```

**마일스톤**:
- 기본 프로그램 컴파일 성공
- 테스트 100개 통과

---

#### Week 9-12: LLVM 통합 + 최적화 기초

**목표**: FreeLang → 기계어 생성 시작

**담당**: CodeGen Agent + Optimizer Agent (병렬)

**CodeGen 작업**:
```
x86-64 코드생성 기초
  ├── 간단한 함수 생성
  ├── 기본 명령어 (mov, add, call)
  ├── 스택 프레임 관리
  └── 심볼 정보
```

**Optimizer 작업**:
```
최적화 패스 1차
  ├── 상수 폴딩
  ├── 데드 코드 제거
  ├── 인라이닝 기초
  └── 성능 측정
```

**마일스톤**:
- 최소 프로그램 컴파일 가능
- Rust 대비 20배 수준 성능

---

### Phase 2: 코드생성 고도화 (Weeks 13-24)

#### Week 13-20: 완전 x86-64 지원

**목표**: 모든 기본 구문 지원, 성능 개선

**담당**: CodeGen Agent + Optimizer Agent

**작업**:
```
고급 x86-64 생성
  ├── 모든 연산자 지원
  ├── 함수 호출 (cdecl/fastcall)
  ├── 메모리 접근 (배열, 구조체)
  ├── 조건분기 최적화
  └── 루프 언롤링
```

**마일스톤**:
- 중간 복잡도 프로그램 컴파일
- 성능 Rust 대비 5배 수준

---

#### Week 21-24: 링커 + 라이브러리 통합

**목표**: 독립 실행파일 생성, 표준 라이브러리 링킹

**담당**: CodeGen Agent + Runtime Agent

**작업**:
```
링커 구현 (linker.fl)
  ├── ELF 포맷 생성
  ├── 심볼 해석
  ├── 재배치 처리
  ├── 동적 링킹
  └── 표준 라이브러리 통합

런타임 라이브러리 (libc 대체)
  ├── printf, malloc, free
  ├── syscall 래퍼
  └── 표준 함수
```

**마일스톤**:
- 독립 .elf 파일 실행 가능
- printf 등 기본 함수 동작

---

### Phase 3: 부트로더 + 커널 기초 (Weeks 25-32)

#### Week 25-28: 부트로더 (Bootloader)

⚠️ **주의**: 이 단계는 어셈블리 필수 (FreeLang 불가)

**목표**: x86-64 부트로더 작성

**담당**: 팀장 + 특화 에이전트

```
bootloader.asm (500줄, x86-64 어셈블리)
  ├── BIOS 초기화
  ├── CPU 레지스터 설정
  ├── 메모리 맵 읽기
  ├── A20 라인 활성화
  ├── 32비트 → 64비트 전환
  ├── 커널 로드
  └── 커널로 점프
```

**마일스톤**:
- 부팅 가능한 USB 생성
- 커널 로드 확인

---

#### Week 29-32: 커널 기초 (Kernel Base)

**목표**: 최소 커널 (메모리, 프로세스 관리)

**담당**: Runtime Agent + CodeGen Agent (협력)

```
kernel.fl (~500줄)
  ├── 메모리 관리 (물리/가상)
  ├── 프로세스/스레드 관리
  ├── 인터럽트 핸들러
  ├── 간단한 스케줄러
  └── 기본 시스템콜
```

**마일스톤**:
- FreeLang으로 컴파일된 프로그램 실행
- 멀티태스킹 기초
